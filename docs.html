language১. ANSI ম্যাপিং কী?
expand_more

আপনি যখন বাংলায় টাইপ করেন — যেমন "সালাম" — আপনার কম্পিউটার ইউনিকোড (Unicode) ব্যবহার করে। কিন্তু পুরনো অনেক ফন্ট (যেমন সোটনমিজে / বিজয়) ইউনিকোড বোঝে না। তারা ANSI এনকোডিং বোঝে।

এই JSON ফাইলটি হল সেই অনুবাদক — এটি বলে দেয় প্রতিটি ইউনিকোড বর্ণের জন্য কোন ANSI কোড ব্যবহার করতে হবে। যেমন:

স (ইউনিকোড) → #$A2 (ANSI কোড)
া (ইউনিকোড) → #$79 (ANSI কোড)
ল (ইউনিকোড) → #$0153 (ANSI কোড)
া (ইউনিকোড) → #$79 (ANSI কোড)
ম (ইউনিকোড) → #$203A (ANSI কোড)

ফলাফল: সালাম (ইউনিকোড) → ‡yv (ANSI সোটনমিজে)

account_tree২. JSON ফাইলের গঠন
expand_more

JSON ফাইলটি ৫টি প্রধান অংশে বিভক্ত:

{
  "Constants":        // বর্ণ ও গ্লিফের সংজ্ঞা
  "FullFormReplacements": // পূর্ণ যুক্তাক্ষরের তালিকা
  "PreReplacements":  // আগাম প্রতিস্থাপন
  "PostReplacements": // পশ্চাৎ প্রতিস্থাপন (সবার শেষে)
  "VowelRules":       // স্বরবর্ণের গ্লিফ নির্বাচনের নিয়ম
  "ConsonantGroups":  // ব্যঞ্জনবর্ণের গ্রুপ
}
Constants → ৭টি উপবিভাগ

Numbers (০-৯): অঙ্কের ANSI ম্যাপিং — "A_0": { "Value": "#$0030", "Comment": "০" }

VowelsAndKars: স্বরবর্ণ ও কারচিহ্ন — "A_AAKar": { "Value": "#$0079", "Comment": "া" }

Symbols: যতি ও প্রতীক — "A_Hasanta": { "Value": "#$E4", "Comment": "্" }

Consonants: ব্যঞ্জনবর্ণ — "A_K": { "Value": "#$201E#$FE", "Comment": "ক" }

FullForms: যুক্তাক্ষর — "A_K_K": { "Value": "#$45#$FE", "Comment": "ক্ক" }

FirstHalfForms: বর্ণের প্রথম খন্ড — "A_Reph": { "Value": "#$C5", "Comment": "র-এর রেফ" }

SecondHalfForms: বর্ণের দ্বিতীয় খন্ড — "A_ZFola": { "Value": "#$C4", "Comment": "য-ফলা" }

FullFormReplacements → পূর্ণ যুক্তাক্ষর 
{ "Key": "#$09A8#$09CD#$09A4", "Value": "#$68#$73#$FE", "Comment": "ন্ত" }
alt_route৩. কনভার্সন প্রক্রিয়া: ধাপে ধাপে
expand_more

ইউনিকোড থেকে ANSI-তে রূপান্তর একটি নির্দিষ্ট ক্রমে ঘটে। নিচে প্রতিটি ধাপ বর্ণনা করা হলো:

সালাম (ইউনিকোড)
    ↓
① FullFormReplacements
    ↓
② PreReplacements
    ↓
③ ReplaceFullForms (FirstHalf + SecondHalf)
    ↓
④ ReplaceKarsVowels (VowelRules দ্বারা)
    ↓
⑤ PostReplacements (#{Var} সম্প্রসারণ)
    ↓
⑥ FinalTouch (গ্লিফ পুনর্বিন্যাস + ANSI পরিশোধন)
    ↓
‡yv (ANSI আউটপুট)
① FullFormReplacements — বিস্তারিত

সবচেয়ে আগে চলে। যুক্তাক্ষর পুরো আকারে ANSI-তে রূপান্তরিত হয়। Key-গুলো দীর্ঘ থেকে ক্ষুদ্র ক্রমে সাজানো থাকে।

ন্ত্ব → #$67#$73#$FE
ন্ত   → #$68#$73#$FE
② PreReplacements — বিস্তারিত

ক্যারেক্টার প্রসেসিংয়ের আগে বিশেষ প্রতিস্থাপন। যেমন হসন্ত+দাঁড়ি → ব্যাকস্ল্যাশ।

্। → #$FF
③ ReplaceFullForms — বিস্তারিত

FirstHalfForms ও SecondHalfForms ব্যবহার করে যুক্তাক্ষর তৈরি। যেমন র-এর রেফ + প = র্প।

④ ReplaceKarsVowels — বিস্তারিত

সবচেয়ে জটিল ধাপ। কারচিহ্ন (ু, ূ, ৃ) বসানোর জন্য VowelRules ব্যবহার করে। নিচে বিস্তারিত দেখুন।

⑤ PostReplacements — বিস্তারিত

সব গ্লিফ বসানোর পর অর্ডারিং ফিক্স। এখানে #{VariableName} ব্যবহার করা যায়।

"Key": "#$FE#{A_UKar1}", "Value": "#{A_UKar1}#$FE"
⑥ FinalTouch — বিস্তারিত

সর্বশেষ ধাপে পুনর্বিন্যাস ও পরিষ্কারকরণ:

    য-ফলা+উ-কার → উ-কার+য-ফলা
    র-ফলা+উ-কার → উ-কার+র-ফলা
    রেফ+উ-কার → উ-কার+রেফ
    অপ্রয়োজনীয় ইউনিকোড (ZWJ, ZWNJ) সরানো
rule৪. VowelRules: স্বরবর্ণের গ্লিফ নির্বাচন
expand_more

VowelRules নির্ধারণ করে — একটি কারচিহ্ন (ু, ূ, ৃ) কোন ব্যঞ্জনবর্ণের পরে বসলে কোন ANSI গ্লিফ ব্যবহার করবে।

প্রথমে: ম্যাপিং (mappings) চেক করে। পূর্ববর্তী ব্যঞ্জনবর্ণ "consonants" তালিকায় থাকলে সেই mapping-এর value ব্যবহার করে।

দ্বিতীয়ত: ম্যাপিং না মিললে কিন্তু বর্ণটি BaseLineRight তালিকায় থাকলে "baselineRight" ব্যবহার করে।

সর্বশেষ: কোনটাই না মিললে "default" মান ব্যবহার করে।

বাস্তব উদাহরণ: উ-কার

"#{b_Ukar}": {
  "default": "#{A_UKar1}",        // #$25 — সাধারণ
  "baselineRight": "#{A_UKar2}",  // #$25 — ঝুলন্ত
  "mappings": [
    { "consonants": "#{b_r}", "value": "#{A_UKar2}", "alt": "#{A_UKar4}" },
    { "consonants": "#{b_L}", "value": "#{A_UKar2}", "alt": "#{A_UKar4}" }
  ]
}

কেস ১: রু — পূর্ববর্তী বর্ণ র (র) mapping-এ আছে → A_UKar2। toggle চালু থাকলে alt = A_UKar4।

কেস ২: গু — গ (গ) mapping-এ নেই, কিন্তু BaseLineRight → baselineRight = A_UKar2।

কেস ৩: ড়ু — ড় (ড়) mapping-এ নেই, BaseLineRight-ও নয় → default = A_UKar1।

variables৫. #{VariableName} — ভ্যারিয়েবল এক্সপ্যানশন
expand_more

সরাসরি হেক্স কোডের পরিবর্তে #{A_UKar1} ব্যবহার করতে পারেন। ইঞ্জিন JSON লোড করার সময় Constants থেকে রেজল্ভ করে।

সংগতি: এক জায়গায় পরিবর্তন করুন, সব জায়গায় আপডেট হবে।

পাঠযোগ্যতা: #{A_UKar2} বোঝা সহজ, #$25 বোঝা কঠিন।

ভার্সনিং: প্রতি AnsiVersion-এর জন্য আলাদা Constants সেট।

উদাহরণ: #$FE#{A_UKar1}
→ #{A_UKar1} রেজল্ভ হয় #$25-এ
→ ফলে #$FE#$25 = দুটি ANSI বাইট
code৬. প্রতিটি ফাংশন কিভাবে কাজ করে
expand_more

ExpandVarRefs

ইনপুট স্ট্রিং-এ #{VarName} প্যাটার্ন খুঁজে Constants থেকে মান এনে বসায়।

লজিক: Pos('#{') → PosEx('}') → AnsiRegistryMap খুঁজে → মান বসায় → আবার খোঁজে

ResolveStringValue

ExpandVarRefs + ProcessHexAndUnicode একসাথে করে।

"#$FE#{A_UKar1}" → ExpandVarRefs → "#$FE#$25" → ProcessHexAndUnicode → দুটি ক্যারেক্টার

GetVowelGlyph

3-স্তরের প্রায়োরিটি: mapping match → baselineRight fallback → default।

BaseLineRightCharacter

ব্যঞ্জনবর্ণ বেসলাইনে বসে কিনা পরীক্ষা করে। বেসলাইনে বসা বর্ণ: খ, গ, ঘ, ণ, থ, দ, ধ, ন, প, ব, ম, য, র, ল, শ, ষ, স, হ, য়।

ReplaceKarsVowels

উ-কার, ঊ-কার, ঋ-কার প্রক্রিয়া করে। Pos দিয়ে কার খুঁজে → পূর্ববর্তী ক্যারেক্টার বের করে → GetVowelGlyph ডাকে → WideStuffString দিয়ে প্রতিস্থাপন করে।

FinalTouch

সর্বশেষ ধাপ: হসন্ত গ্লিফ ফিক্স, য-ফলা+উ-কার অর্ডার, র-ফলা+উ-কার অর্ডার, ANSI পরিশোধন (শুধু 0x00-0xFF) রাখে।

preview৮. পূর্ণ উদাহরণ: "গু" (গ+উ-কার) থেকে ANSI

① শুরু: ইনপুট = গ (গ) + ু (উ-কার)

② FullFormReplacements: গু এর জন্য এন্ট্রি নেই → skip

শেষ ফলাফল: গ + #$25 = গু (ANSI)
